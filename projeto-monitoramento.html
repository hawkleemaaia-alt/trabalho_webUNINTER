Sistema de Alerta Rural

Case técnico — Back-end, APIs e geolocalização para operação de monitoramento com acionamento por botão de pânico.

Ver contribuições Stack: Node.js • Express • MySQL • Postman • Heroku • E-mail

Contexto

Projeto voltado a patrulhamento em áreas rurais, conectando propriedades a unidades de campo por meio de um painel web e APIs com geolocalização e priorização por proximidade. Fluxo principal: alerta ➜ roteamento por distância ➜ notificação.

Alerta imediato: botão de pânico para registrar evento e iniciar notificação às unidades compatíveis.
Acionamento simples, latência baixa e confirmação de recebimento.
Geolocalização: cálculo de proximidade e despacho orientado por distância/tempo.
Considera localização corrente e status de disponibilidade.
Cenário rural: cadastros de áreas/propriedades e histórico de ocorrências.
Foco em rastreabilidade e resposta rápida.

Tecnologias

  • Back-end: Node.js (migração após tentativa inicial em Python/Pandas)
  • Framework: Express
  • Banco: MySQL
  • Integração: Endpoints HTTP para o front-end (rotas/serviços)
  • Documentação/Testes: Postman
  • E-mail: Disparo transacional (ex.: Nodemailer/SMTP)
  • Hospedagem: Heroku
  • Python \(fase inicial\): Pandas para manipulação de dados (onde aplicável)

Motivação da stack: alinhamento com o front-end em JavaScript e facilidade de integração.
